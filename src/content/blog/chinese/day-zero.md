Robavionix 这周不会发布任何东西。没有硬件,没有课程视频,还没有一个学生碰过故障注入开关。昨天发生的事情比这些都小,但依然值得记下来:域名注册好了,GitHub 仓库建起来了,网站也接通 Cloudflare Pages 挂到了真实域名上。

## 实际发生了什么

这个网站最初是一套通用的 IT 服务主题模板——每个页面都在讲“云咨询”和“数字化转型”,一个买家现成的闪电图标 logo,一支十二个人的假团队,配着网络图库照片。经过一次很长的会话,它变成了现在这样:详细写出来的 L1~L5 课程体系,一个真实(但刻意隐去了具体身份信息)的创始人页面,一个围绕产品核心——滑模控制——设计出来的自有 logo,以及一个中英双语的站点,而不是默认的英法双语。

仓库在 GitHub 上,公开的。部署在 Cloudflare Pages 上,每次 push 到 `main` 都会自动构建。域名已经生效。这就是全部——枯燥但必需的基础设施,本来就应该在跑起来之后就被人遗忘,而这正是为什么现在值得花一段话记下它,趁它还没被遗忘。

## 为什么要把这种事写下来

老实说,一段容错控制的博士研究不会自己变成一款上线的教学产品,而这个故事将来真正值得读的版本,不会是从“然后我们卖出了 200 套”开始讲起的——而是那个还留着枯燥的第一次 commit 的版本。如果你打算公开搭建一样东西,网站上线这天就该算作第 0 天,哪怕第 0 天基本上只是一堆配置文件。

## 接下来真正要做的事

真实的硬件物料清单、第一个跑在物理测试台上的 PID 控制器,以及——按照已经写好的[课程体系](/curriculum/)——比这一篇多得多的博客文章。这是一个开始,不是一个里程碑。
